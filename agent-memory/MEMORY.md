# Memory — graphify

## Active Decisions

- [2026-07-03] **God-node ranking excluded import/module anchor nodes (`type in ("module", "namespace")`).** `analyze.py::god_nodes()` ranked by raw degree only filtered file-level hubs, JSON-key noise, and a Python-specific builtin list — so a cross-language import anchor (#1327: `import Foundation` from N files collapses to one shared `type=module` node) accumulated one edge per importing file and mechanically outranked real abstractions. Confirmed on the harness-terminal graph: `Foundation`/`HarnessCore` topped its god-node list ahead of `SessionCoordinator`/`SessionEditor`. Fix: exclude any node whose `type` attribute is `module`/`namespace` — the same tag `dedup.py`'s module-anchor collapse already relies on (#1327), so this reuses an existing, precise signal instead of adding a new heuristic. Regression test: `test_god_nodes_excludes_module_anchor_nodes` in `tests/test_analyze.py`.
- [2026-07-03] **Per-project configuration (`graphify.toml` / `[tool.graphify]` in `pyproject.toml`) + native `update --all` batch command.** `graphify/config.py::load_project_config()` loads project-local defaults (resolution, exclude_hubs, mode, wiki, no_viz, rank_by, etc.) with precedence graphify.toml > pyproject.toml > explicit CLI args. `update --all` (or `update-all`) discovers projects either via explicit search paths or the global manifest, skips clean repos via `_is_git_fresh()` (compares `git rev-parse HEAD` against `graph.json`'s `built_at_commit`), and per-project either runs a **custom refresh script** (`scripts/graphify-refresh.sh` / `Scripts/graphify-refresh.sh`, auto-delegated if present — the escape hatch for flags config can't express) or falls back to a native Python rebuild driven by that project's config file. `9arm-skills`-named directories are always excluded. `--wiki`/`--no-viz` are now also available directly under `extract` (previously only under `update`/config). Full test suite: 2890 passed, 28 skipped. **Lesson: the script lives in each individual project, not embedded in graphify itself — graphify only embeds the discovery/delegation logic that looks for it.**
- [2026-07-03] **Local repo installed in editable mode (`uv tool install --editable . --force`), replacing the stale PyPI 0.10.0 install.** The globally installed `graphifyy` CLI had drifted 4 minors behind this repo (0.10.0 vs 0.14.0+uncommitted), causing a version-skew warning on every invocation. Editable install means local edits (like the two decisions above) are immediately live via the `graphify` command in every other project, without a reinstall step. Skill re-synced via `graphify install`.
